# 数据分析课程卡片重新设计 - 产品需求文档

## 概述
- **总结**: 重新设计5门核心数据分析课程的卡片展示，增加模块拆分功能，让用户能够清晰了解每门课程的学习路径和内容结构
- **目的**: 解决原课程卡片"无模块拆分、无法逐一学习"的问题，提供更有结构感的课程展示，增强用户学习体验
- **目标用户**: 数据分析学习者、职业发展者、学生和在职人员

## 目标
1. 为5门核心课程添加清晰的模块结构信息
2. 重新设计课程卡片展示，突出模块信息
3. 优化信息层级：课程名称 > 阶段标签 > 模块列表 > 课时数据
4. 确保风格统一，适合网页端展示
5. 移除卡片上的乱码英文，使用主题相关的封面图标

## 非目标（范围外）
- 不修改现有课程内容和练习
- 不重构整个课程详情页面
- 不添加新的功能（除模块展示外）
- 不修改其他课程（6号和7号课程）

## 背景与上下文
- 当前课程系统已有7门课程，其中5门需要重新设计
- 现有课程卡片只显示基本信息（标题、描述、时长、节数）
- 缺少模块信息，用户无法快速了解课程结构
- 项目使用 React 18 + TypeScript + Tailwind CSS + Zustand

## 涉及的5门课程
1. **Python 数据分析基础**（入门）- 拆分为3个模块，每个模块5节课，单节20分钟，总课时20小时
2. **Excel 数据处理与分析**（入门）- 拆分为3个模块，每个模块4节课，单节15分钟，总课时15小时
3. **数据可视化实战**（中级）- 拆分为3个模块，每个模块6节课，单节25分钟，总课时25小时
4. **统计分析与假设检验**（中级）- 拆分为3个模块，每个模块7节课，单节30分钟，总课时30小时
5. **机器学习入门**（高级）- 拆分为3个模块，每个模块8节课，单节30分钟，总课时40小时

## 功能需求
### FR-1: 课程数据结构扩展
- 在 `Course` 类型中添加 `modules` 字段，支持模块信息
- 每个模块包含：模块名称、课时数量、课程列表
- 更新5门目标课程的数据，添加完整的模块信息

### FR-2: 课程卡片重新设计
- 重新设计课程卡片UI，展示模块列表
- 突出显示课程名称、阶段标签
- 清晰展示模块信息（模块名称 + 课时数量）
- 优化信息层级和视觉呈现
- 保持响应式设计，适配不同屏幕尺寸

### FR-3: 封面图标优化
- 为每门课程生成主题相关的高质量封面图
- 确保图片与课程主题强相关
- 使用清晰的视觉设计

## 非功能需求
- **NFR-1**: 设计风格统一，配色干净清爽
- **NFR-2**: 响应式设计，适配各种设备尺寸
- **NFR-3**: 保持现有功能（搜索、筛选、进度显示）
- **NFR-4**: 加载性能不受明显影响

## 约束条件
- **技术**: 必须使用现有的技术栈（React 18, TypeScript, Tailwind CSS）
- **兼容性**: 保持与现有代码的向后兼容性
- **范围**: 仅修改5门目标课程和相关组件

## 假设
- 现有课程功能（学习、练习、进度跟踪）无需修改
- 用户希望按模块学习课程
- 课程数据结构的修改不会影响现有系统功能

## 验收标准
### AC-1: 模块数据结构正确
- **前提**: 项目已更新课程类型定义
- **当**: 查看课程数据文件
- **则**: 5门目标课程均有完整的 `modules` 字段，包含正确的模块名称、课时数
- **验证类型**: programmatic

### AC-2: 课程卡片显示模块信息
- **前提**: 访问课程列表页面
- **当**: 查看5门目标课程的卡片
- **则**: 卡片清晰显示模块列表，每个模块标注名称和课时数
- **验证类型**: human-judgment

### AC-3: 信息层级正确
- **前提**: 查看课程卡片
- **当**: 浏览课程卡片内容
- **则**: 信息按以下顺序呈现：课程名称 > 阶段标签 > 模块列表 > 课时数据
- **验证类型**: human-judgment

### AC-4: 设计风格统一
- **前提**: 查看全部课程卡片
- **当**: 比较不同课程卡片的设计
- **则**: 整体风格统一，配色干净清爽，适合网页端展示
- **验证类型**: human-judgment

### AC-5: 响应式设计正常
- **前提**: 使用不同设备尺寸访问
- **当**: 调整浏览器窗口大小或使用移动设备
- **则**: 课程卡片在各种屏幕尺寸下都能正常显示，布局合理
- **验证类型**: human-judgment
